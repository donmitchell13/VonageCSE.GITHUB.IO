VONAGE VERIFY API TO SALESFORCE

At Vonage, we focus on making our API experience as easy as possible. The flexibility of APIs allows you to build combinations of complementary technologies—for example, the Salesforce CRM and the Vonage Verify API (https://developer.nexmo.com/api/verify). In this tutorial, I want to take you on an adventure to incorporate the Verify API into Salesforce with a focus on marketing leads. In the process, we’ll touch on several aspects of Salesforce (apex, triggers, visual force pages ) from the declarative methods (process builder, site.com, and platform cache).

This may not need explaining, but a project like this can make you a hero to the Marketing and Sales teams. Just imagine, tracking leads with actual verified phone numbers that can be dispensed to the Sales or Sales Development Representative without the dreaded cold calling.  

So back to reality—how do we get this wonderful bit of API calls up and running?  

Prerequisites

  • A Salesforce sandbox or Salesforce developer instance.  
  • System administrators access or enough access to make the following updates and request in the instance.

Look around and get familiar with the Vonage Verify API.  This unmanaged package is focused on only two aspects of the API, Request a Verification and Verify Check. In short,” request a verification” can be thought of as an exchange with the prospect of their cell number for a security code. “Verify Check” determines if the code returned by a prospect is valid—this is GOLD to the Marketing team, as it confirms the number you have is legitimate.
